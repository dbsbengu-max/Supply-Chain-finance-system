## 3. 常用操作流程

### 3.1 登录与切换身份

1. 打开 `/login`，输入账号密码。  
2. 进入工作台后，在「当前身份」下拉切换运营商/项目身份。  
3. 切换后列表数据按项目范围刷新。

### 3.2 贸易代采

1. **贸易代采** → **新建**。  
2. 填写必填项 → **保存草稿** → **提交**。  
3. 列表/详情确认状态为 `SUBMITTED`。

### 3.3 融资与放款

1. 使用 `funding_user` 登录（或 platform_admin 审批后切换）。  
2. **融资管理** → 选择单据 → **审批** → **放款前校验** → **放款**。  
3. 确认状态 `DISBURSED`。

### 3.4 清分

1. **清分中心** → 选择融资单/流水。  
2. **试算** → 确认金额 → **执行清分**。  
3. 结果状态 `EXECUTED`。

### 3.5 签章（Mock / HTTP Adapter）

1. **签章中心** → 登记单证 → **发起签署**。  
2. 跟踪签署状态；异常单在 **补偿池** 筛选 `CONTRACT_SIGN_CALLBACK` 人工处理。

### 3.6 补偿池人工处理

1. **补偿池** → 补偿任务 Tab。  
2. 认领/查单/重试须填写**人工原因**（写入审计日志）。  
3. 签章回调类任务默认 **人工复核**，勿期望全自动重试成功。

## 6. 故障排查

| 现象 | 处理 |
|---|---|
| 登录 401 | 确认后端 8080 UP；JWT 环境变量与 prod profile 一致 |
| 菜单 403 | 换 `platform_admin` 或查 [`TEST_ACCOUNT_MATRIX.md`](TEST_ACCOUNT_MATRIX.md) |
| 列表空 | 执行 demo seed |
| 签章失败 | 使用 Mock 提供商；查补偿池而非真实 vendor |
| 后端无法启动 | `mvn clean package` 打 fat jar；Java 用 D 盘 JDK（C 盘 Temurin 可能损坏） |
