## 3. 建议下一轮任务

### Cursor

1. 完成“功能上线收口页面”：系统配置、供应商配置、补偿池、签章中心、BI 看板、试点闭环入口的菜单与空态。
2. 补齐试点演示路径：客户/KYC → 贸易代采 → 仓储货权 → 融资放款 → 清分 → 签章 → BI。
3. 为每个关键页面补统一错误提示、金额格式、权限隐藏、禁用原因和操作后反馈。
4. 维护一套演示 Mock Data，确保重新初始化后可重复演示。

### Codex

1. 做一轮后端回归，确认最新迁移、权限、Saga、签章、补偿池没有破坏既有用例。
2. 复核前端 smoke 和路由权限，形成 EA-049 验收结果。
3. 固化 UAT 最小集：登录、客户、贸易、仓储、融资、清分、签章、BI、补偿池、权限穿透。
4. 输出上线剩余缺口清单，区分“功能上线阻断”和“真实生产阻断”。

## 4. 验收口径

| 维度 | 最低通过标准 |
|---|---|
| 功能完整性 | 核心链路可从页面发起并闭环，关键操作有状态变化和审计记录 |
| 权限安全 | 平台、企业、资金方、仓储、审计等角色不能越权访问跨项目数据 |
| 数据一致性 | 订单、库存、融资、清分、签章状态联动可追踪，异常进入补偿或人工处理 |
| 可演示性 | 使用固定账号和 Mock Data 可在本地/试点环境重复演示 |
| 可运维性 | 健康检查、补偿池、错误日志、证据导出可用于定位问题 |
| 后置边界 | 文档明确标注真实 vendor、真实银行、生产灰度仍为后置待办 |
