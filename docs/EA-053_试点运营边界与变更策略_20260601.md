# EA-053 试点运营边界与变更策略

**生效**：EA-052 GO trial 之后  
**原则**：试点运营 = 稳态 + 反馈；**EA-048 / 生产 = 独立专项**

---

## 1. 策略总览

```
试点运营（EA-053）          生产专项（EA-048+）
─────────────────          ─────────────────
P0 bugfix                  真实 vendor API
文档 / 脚本                真实银行 SDK
巡检 / 反馈台账            ALLOWLIST 灰度
demo seed                  Staging/Pilot 主机
run-ea051 / smoke          Go/No-Go 签核

         ╳ 不交叉混 PR ╳
```

---

## 2. 冻结延续（EA-051 + EA-053）

| 维度 | 试点运营期 |
|---|---|
| 功能 scope | **FROZEN**（与 EA-051 相同） |
| Agent 改代码 | 仅 P0 + 测试/脚本 |
| 新 Epic | BI 深化、新模块、体验大改 → **新开**，不得挂 pilot hotfix |
| EA-048 代码路径 | **冻结勿动**（见 EA-051 §4） |

---

## 3. PR / 分支命名

| 类型 | 前缀 | 示例 |
|---|---|---|
| P0 修复 | `fix(pilot-p0):` | `fix(pilot-p0): clearing 403 for admin` |
| 运营文档/脚本 | `chore(pilot-ops):` | `chore(pilot-ops): update patrol checklist` |
| 用户反馈（仅文档） | `docs(pilot-feedback):` | 更新 USER_FEEDBACK_LOG |
| **禁止** | `feat(ea-048):` 在 pilot 分支 | → 必须 EA-048 专项分支 |

合并前：**P0 必跑** `run-ea051-acceptance.ps1`；文档-only 可 `-SkipRegression`。

---

## 4. 需求分流

| 用户说法 | 分流 |
|---|---|
| 「演示时代采提交报错」 | P0 评估 → pilot hotfix |
| 「想接 e签宝真环境」 | EA-048 台账，**不** pilot hotfix |
| 「BI 要能导出 Excel」 | backlog / 新 Epic |
| 「补偿池想自动处理 CALLBACK」 | 非 P0 → 反馈台账 |
| 「Staging 域名怎么配」 | DEF-STG / 运维文档 |

---

## 5. Agent（Cursor / Codex）指令

1. 改代码前读 EA-051 + 本文件  
2. 若 ticket 含 vendor / 银行 / 灰度 / prod → **停止**，只更新 DEF-048 登记  
3. P0 fix 后跑 A1–A4  
4. 禁止「顺手」改 UI、加字段、扩 API
