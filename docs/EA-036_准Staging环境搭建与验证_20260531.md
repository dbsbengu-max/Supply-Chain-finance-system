# EA-036 准Staging环境搭建与验证

日期：2026-05-31

## 1. 背景

当前没有跳板机，也没有 staging 后端机，因此 EA-035 无法直接执行“真实 staging 签核”。为避免发布前验证停摆，新增 EA-036：用 Docker Compose 建立可复现的准 Staging 环境，作为 EA-035 真实跑闸前的技术预验证载体。

## 2. 范围

| 项目 | 说明 |
|---|---|
| PostgreSQL | 独立容器，host 端口默认 `15432` |
| 后端 | Spring Boot，`prod` profile，host 端口默认 `18080` |
| 前端 | Nginx 静态服务，host 端口默认 `15173` |
| 用户密码 | 四个试点用户重置为 UAT 手册密码，仅用于准 staging |
| 验证脚本 | 复用 EA-035 `run-staging-gate.ps1` |

## 3. 交付物

| 文件 | 用途 |
|---|---|
| `deploy/quasi-staging/docker-compose.yml` | 准 staging 编排 |
| `deploy/quasi-staging/backend.Dockerfile` | 后端镜像构建 |
| `deploy/quasi-staging/frontend.Dockerfile` | 前端镜像构建 |
| `deploy/quasi-staging/nginx.conf` | 前端反向代理 `/api/v1` |
| `deploy/quasi-staging/.env.example` | 准 staging 环境变量模板 |
| `deploy/quasi-staging/seed/quasi_passwords.sql` | 演示密码重置 |
| `deploy/quasi-staging/scripts/up.ps1` | 启动 |
| `deploy/quasi-staging/scripts/verify.ps1` | 执行 EA-035 gate |
| `deploy/quasi-staging/scripts/down.ps1` | 停止 |

## 4. 执行步骤

```powershell
cd deploy\quasi-staging
copy .env.example .env
.\scripts\up.ps1 -Rebuild
.\scripts\verify.ps1 -WatchMinutes 30 -WatchIntervalMinutes 5
```

访问地址：

| 服务 | 地址 |
|---|---|
| 前端 | `http://localhost:15173` |
| 后端健康检查 | `http://localhost:18080/api/v1/actuator/health` |
| PostgreSQL | `localhost:15432/scf` |

## 5. 验收标准

| 验收项 | 标准 |
|---|---|
| 容器启动 | postgres/backend/web 均 running，backend health=UP |
| Flyway | 迁移到 `>= v1.1.027` |
| seed verify | 四用户 ACTIVE，关键权限存在，`password_hash != mock_hash` |
| A-01 | health URL 返回 UP |
| A-03 | outbox FAILED=0 或 StrictStale 无 stale 失败 |
| A-04 | compensation MANUAL_REQUIRED=0 |
| 前端 | 登录页可打开，登录后可进入工作台 |

## 6. 边界与限制

准 Staging 不是生产发布签核环境，不能替代真实 staging。原因：

- 使用本地/临时 Docker 网络；
- 使用演示密码；
- 密钥为准 staging 模板值；
- 未覆盖真实网关、域名、TLS、WAF、备份、监控、权限审批链路。

## 7. 后续推进

1. 用 EA-036 跑通准 staging，修掉容器化、迁移、seed、health、脚本级问题。
2. 申请或搭建真实 staging/pre-prod 主机。
3. 将 EA-036 中稳定的 compose/Dockerfile 升级为正式部署流水线基础。
4. 在真实环境重跑 EA-035，并完成 Go/No-Go 签字。

## 8. 无Docker兜底路径

如果当前机器没有 Docker，但有本地 PostgreSQL 和 Java 17，可以使用 `deploy/pilot/scripts/run-local-prod-rehearsal.ps1` 做 prod profile 预演：

```powershell
cd deploy\pilot
copy .env.local-prod-rehearsal.example .env.local-prod-rehearsal
.\scripts\run-local-prod-rehearsal.ps1 -WatchMinutes 5 -WatchIntervalMinutes 1
```

该脚本优先使用已构建 jar；没有 jar 时尝试 Maven；没有 Maven 时尝试 `target/classes + target/cp-jars`。它可以验证 prod profile、Flyway、seed、health、A-03/A-04 脚本链路，但仍不能替代真实 staging Go/No-Go。

## 9. 裁决

EA-036 是当前无跳板机、无 staging 后端机条件下的最优下一步。它可以把“环境缺失”从不可执行问题，转化为一个可验证、可复现、可移交运维的部署包。
