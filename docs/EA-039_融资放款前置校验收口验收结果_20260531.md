# EA-039 融资放款前置校验收口验收结果

生成日期：2026-05-31

## 1. 验收结论

**通过。** EA-039 将单证中心 `/documents/validate` 能力接入融资放款主链路，提供 `POST /finance/applications/{id}/pre-check` 统一前置校验，并在 `POST .../disburse` 执行前强制门禁，覆盖单证齐备、OCR/人工复核、授信额度、账户余额、BPM 状态、幂等与二次确认。

本次交付覆盖：

- 数据库迁移 `V1_1_030__finance_disburse_pre_check.sql`：新增权限 `FINANCE_PRECHECK` 并授权 `ROLE_PLATFORM_ADMIN`、`ROLE_FUNDING`。
- 后端 `FinancePreCheckService`：编排 8 类检查项，内部复用 `DocumentValidationService.validateFinanceDisburse()`（与 `/documents/validate` 同源规则）。
- `FinanceApplicationService.doDisburse()`：在状态/金额校验通过后调用 `assertDisbursePreCheck()`，未通过则 `FINANCE_PRECHECK_409`。
- 幂等哈希与放款接口一致：门禁使用与 `IdempotencyService` 相同的 request body 计算 SHA-256，避免 PROCESSING 记录误报冲突。
- 前端融资列表抽屉「放款前校验与执行」：先 pre-check 展示检查表，通过后方可放款。
- 集成测试 `FinancePreCheckIntegrationTest`（7 用例）纳入 `EA019RegressionRunner`；EA-015 放款回归保持通过。

## 3. 前置校验项

| code | 说明 | 结果级别 |
|---|---|---|
| `FINANCE_STATUS` | 须为 `TO_DISBURSE` | FAILED 阻断 |
| `BPM_APPROVED` | BPM 实例 `COMPLETED`/`APPROVED`；无实例则视为平台审批已通过 | FAILED 阻断 |
| `CREDIT_AVAILABLE` | 关联授信有效期、币种、可用额度；未关联授信为 WARNING | FAILED / WARNING |
| `DOCUMENTS` | 复用单证中心规则（`FINANCE` + `DISBURSE` + product_type） | FAILED 阻断 |
| `PAYER_ACCOUNT` | 出款户存在、归属、余额 | FAILED 阻断 |
| `RECEIVER_ACCOUNT` | 收款户存在、归属 | FAILED 阻断 |
| `IDEMPOTENCY_KEY` | 幂等键存在且与历史 hash 一致 | FAILED / WARNING |
| `SECONDARY_AUTH` | 二次确认令牌（pre-check 接口）；disburse 门禁使用已通过的外层校验 | FAILED / WARNING |

响应结构（与 PRD 契约一致）：

```json
{
  "passed": true,
  "checks": [
    { "code": "DOCUMENTS", "result": "PASSED", "message": "..." }
  ]
}
```

## 4. 与单证中心衔接

- `AGENCY_PURCHASE` 融资放款阶段须具备 `PURCHASE_CONTRACT`、`INVOICE` 且人工复核通过（规则来自 V1_1_029）。
- `ORDER_FINANCE` 等产品若无 DISBURSE 阶段规则，单证项自动通过。
- pre-check 响应附带完整 `DocumentValidateResponse`，便于前端展示 missing / pending / warnings 明细。

### FinancePreCheckIntegrationTest 用例摘要

1. 齐备单证 + 额度 + 余额 → `passed=true`
2. 缺失必备单证 → `DOCUMENTS=FAILED`
3. 授信不足 → `CREDIT_AVAILABLE=FAILED`
4. 出款户余额不足 → `PAYER_ACCOUNT=FAILED`
5. pre-check 失败时 disburse → `FINANCE_PRECHECK_409`
6. 成员企业无权 pre-check → `AUTH_403`
7. 无二次确认令牌 → `SECONDARY_AUTH=WARNING`（pre-check 仍可 passed）

## 6. 已知限制（V1.1 MVP）

- `fn_finance_application` 尚未持久化 `bpm_instance_id`；BPM 检查为可选 lookup，无实例时不阻断。
- pre-check API 与 disburse 对幂等 body 的字段集合略有差异（pre-check 无 remark）；实际放款门禁使用 disburse 完整 body hash。
- 授信占用在放款成功后仍由原有 disburse 逻辑处理，pre-check 仅做可用额度只读校验。

## 7. 结论

单证中心已接入金融主链路：放款前可统一查看并拦截单证、额度、账户与流程风险，满足 EA-039「融资放款前置校验收口」交付目标。
