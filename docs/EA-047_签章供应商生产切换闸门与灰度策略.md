# EA-047 签章供应商生产切换闸门与灰度策略

**日期**：2026-06-01  
**前置**：EA-045 真实供应商集成包、EA-046 Sandbox 证据包 **PASS**

## 目标

在不一次性全量切换的前提下，将合同签署从 `MOCK` 逐步路由到生产 `ESIGN_HTTP`，并提供可观测、可回滚的切换闸门。

## 灰度模式（`SCF_CONTRACT_SIGN_ROLLOUT_MODE`）

| 模式 | 行为 | 典型用途 |
|---|---|---|
| `OFF` | 忽略 rollout，使用 `SCF_CONTRACT_SIGN_DEFAULT_PROVIDER` | 开发 / 未启用灰度 |
| `ALLOWLIST` | 仅 `project_allowlist` / `operator_allowlist` 命中时使用生产供应商 | 试点项目首批 |
| `PERCENT` | `hash(projectId) % 100 < project_hash_percent` 时使用生产供应商 | 按比例放量 |
| `FULL` | 全量走生产供应商（仍校验 HTTP 配置） | 全量切换 |

未命中灰度时使用 `fallback_provider`（默认 `MOCK`）。

## 环境变量（prod profile）

| 变量 | 说明 |
|---|---|
| `SCF_CONTRACT_SIGN_ROLLOUT_MODE` | OFF / ALLOWLIST / PERCENT / FULL |
| `SCF_CONTRACT_SIGN_ROLLOUT_PRODUCTION_PROVIDER` | 默认 `ESIGN_HTTP` |
| `SCF_CONTRACT_SIGN_ROLLOUT_FALLBACK_PROVIDER` | 默认 `MOCK` |
| `SCF_CONTRACT_SIGN_ROLLOUT_PROJECT_ALLOWLIST` | 逗号分隔 projectId |
| `SCF_CONTRACT_SIGN_ROLLOUT_OPERATOR_ALLOWLIST` | 逗号分隔 operatorId |
| `SCF_CONTRACT_SIGN_ROLLOUT_PROJECT_HASH_PERCENT` | 0–100 |
| `SCF_CONTRACT_SIGN_ROLLOUT_REQUIRE_HTTP_CONFIGURED` | 生产路由前要求 HTTP 供应商 configured |
| `SCF_CONTRACT_SIGN_ROLLOUT_BLOCK_WHEN_MISCONFIGURED` | 未配置时返回 503 或回退 MOCK |

模板见 `deploy/pilot/.env.prod-cutover.example`。

## 可观测性

`GET /integrations/contracts/sign/config` 响应新增 `production_rollout`：

- `mode` / allowlist / percent
- `effective_provider_for_context` — 当前请求上下文将使用的供应商
- `routed_to_production` — 是否命中生产路由

## 推荐切换步骤

1. **闸门 0**：EA-046 sandbox 证据 **PASS**（含 trace / callback / 补偿池）
2. **阶段 1 — ALLOWLIST**：仅 1 个试点 `PJ_*`，观察 24h 签署成功率与补偿池
3. **阶段 2 — PERCENT**：5% → 25% → 50% → 100%
4. **阶段 3 — FULL**：关闭 MOCK 回退路径（保留配置以便紧急回滚）
5. 每阶段执行 `run-ea047-prod-cutover-gate.ps1`，全部 PASS 再放量

## 回滚

1. 将 `SCF_CONTRACT_SIGN_ROLLOUT_MODE=OFF` 或 `ALLOWLIST` 清空 / `PERCENT=0`
2. 重启实例（或滚动发布）
3. 验证 config 中 `routed_to_production=false` 且新发起走 `MOCK`
4. 已在途的 `ESIGN_HTTP` 任务继续走查单 / 回调 / 补偿池，不强制中断
