# EA-030 试点闭环页面与运营体验加固

生成日期：2026-05-27

## 1. 目标

在 EA-029 试点向导与 Saga 监控台基础上，加固**运营侧页面跳转与状态展示**，使代采 → Saga → 融资 → 清分 → BI 形成可追踪的闭环体验。

## 2. Cursor 交付物

### 2.1 代采详情 · 试点闭环时间线

- 文件：`AgencyPurchaseDetailView.vue`、`PilotClosureTimeline.vue`、`utils/pilotClosureTimeline.ts`
- 能力：
  - 端到端时间线（代采 → BPM → Saga → 融资 → 凭证 → 清分 → BI）
  - 关联融资单状态 Tag + 快捷入口（试点向导带 `agency_id`、Saga 台、融资、清分、BI、审计）
  - 补偿任务「Saga 台」深链：`/saga/ops?tab=compensation&business_id=&task_id=`

### 2.2 试点向导 · 上下文加载

- 文件：`PilotClosureView.vue`
- 查询参数：`?agency_id={代采单ID}` 时加载代采详情 + 关联融资，渲染 `PilotClosureTimeline`，步进高亮随时间线进度

### 2.3 Saga 监控台 · URL 深链

- 文件：`SagaOpsCenterView.vue`
- 查询参数：
  - `tab=outbox|compensation`（默认 compensation）
  - `business_id` 预填业务 ID 过滤
  - `task_id` 自动打开补偿任务详情抽屉

### 2.4 融资 / 清分 / BI 跳转与状态

| 页面 | 增强 | 深链参数 |
|---|---|---|
| `FinanceListView.vue` | 状态 Tag、代采来源跳转、清分/BI 快捷、筛选横幅 | `highlight`、`source_id` |
| `ClearingWorkbenchView.vue` | 进入时预选融资单并加载台账 | `finance_id` |
| `BiDashboardView.vue` | 试点上下文横幅 + 返回向导 | `from=pilot` |

### 2.5 字典与组件

- `constants/financeDict.ts` — 融资状态中文与 Tag 类型、`FINANCE_REPAYABLE_STATUSES`

### 2.6 工程卫生

- 执行 `git rm --cached frontend/scf-web/tsconfig.tsbuildinfo`，停止跟踪构建缓存（`.gitignore` 已含 `**/tsconfig.tsbuildinfo`）

## 3. Codex 下一轮复核清单

> **指派 Codex**：权限穿透、金额展示、错误提示、浏览器 smoke、前端路由拆包建议。

### 3.1 页面权限（须 PASS）

| ID | 页面/操作 | 权限码 | 预期 |
|---|---|---|---|
| P-01 | 试点向导 | 各步 `permission` | 无权限显示「无权限」，不可跳转 |
| P-02 | Saga 监控台 | `SAGA_OPS_VIEW` / `SAGA_OPS_MANAGE` | 无 VIEW 不可进路由；无 MANAGE 隐藏重试/批准 |
| P-03 | 融资列表操作 | `FINANCE_VIEW` + 提交/审批后端鉴权 | 只读角色无提交/审批按钮或 API 403 |
| P-04 | 清分试算/执行 | `CLEARING_VIEW` / `CLEARING_EXECUTE` | 无 EXECUTE 不可确认执行 |
| P-05 | BI 导出 | `BI_EXPORT` | 无权限隐藏导出 |
| P-06 | 审计快捷入口 | `AUDIT_VIEW` | 代采详情无权限不显示审计按钮 |

### 3.2 金额展示（须 PASS）

| ID | 场景 | 检查点 |
|---|---|---|
| M-01 | 代采时间线 | `total_amount` + `currency` 与详情一致 |
| M-02 | 融资列表 | `apply_amount` + `currency` 列完整 |
| M-03 | 清分试算结果 | `formatMoney` 与币种一致 |
| M-04 | BI KPI | 与 `formatMoney(overview.*)` 一致，无裸数字 |

### 3.3 错误提示（须 PASS）

| ID | 场景 | 预期 |
|---|---|---|
| E-01 | 代采/融资 API 失败 | `ElMessage.error` 含后端 `message` |
| E-02 | Saga 抽屉加载失败 | 抽屉内或 banner 展示错误，不白屏 |
| E-03 | 清分无融资单 | 「请先选择融资单」warning |
| E-04 | 试点 `agency_id` 无效 | 明确错误提示，向导仍可用 |

### 3.4 浏览器 Smoke（建议顺序）

1. 代采详情 → 时间线各节点跳转 → 返回详情
2. 补偿任务「Saga 台」→ 抽屉自动打开 → 填原因重试（MANAGE 角色）
3. `/finance/applications?highlight={id}` 行高亮
4. `/accounts/clearing?finance_id={id}` 预选融资
5. `/bi/dashboard?from=pilot` 横幅 + 返回向导
6. `/pilot/closure?agency_id={id}` 上下文卡片

### 3.5 前端路由拆包建议（Codex 输出）

当前 `npm run build` 主包约 **1.8MB**（gzip ~602KB），Vite 已提示 chunk 过大。建议 Codex 在复核报告中给出：

- 按域 lazy route：`/bi/*`、`/saga/*`、`/agency-purchase/*`、`/accounts/*`
- ECharts / 大图表组件异步 import
- `manualChunks`：`element-plus`、`echarts`、`vue-vendor`
- 试点页是否独立 chunk（运营低频）
