## 配置要点

```yaml
scf.contract.sign.http-provider:
  outbound-auth-mode: HMAC_SHA256   # 或 RSA_SHA256 / SM2
  platform-trace-header: X-Request-Id
  vendor-request-id-header: X-Vendor-Request-Id
  vendor-trace-id-header: X-Vendor-Trace-Id
  field-mapping:
    task-id: bizTaskNo              # 按供应商文档覆盖
  response-external-ref-field: external_sign_ref
  response-request-id-field: request_id
  response-trace-id-field: trace_id
```

生产环境变量：`SCF_CONTRACT_SIGN_HTTP_OUTBOUND_AUTH_MODE`、`SCF_CONTRACT_SIGN_HTTP_PRIVATE_KEY_PEM`（RSA/SM2）等，见 `application-prod.yml`。

## 备注

- Flyway `V1_1_034` 采用逐列 `ALTER TABLE`（H2 兼容），索引不含 partial `WHERE` 子句。
- 通用 HTTP Adapter 保持单一 `ESIGN_HTTP`；若供应商 canonical/字段差异大，按 `ESIGN_PROVIDER_TEMPLATE.md` 新建 `{Vendor}ContractSignProvider`。
- SM2 依赖 BouncyCastle（`bcprov-jdk18on`）；联调前确认供应商接受 SM3withSM2 + Base64 签名格式。单元测试覆盖 HMAC/RSA，SM2 联调时按 checklist 用真实密钥验签。
