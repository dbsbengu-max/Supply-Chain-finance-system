## 架构说明

- **开发/测试**：`default-provider=MOCK`，`http-provider.enabled=false`（默认）
- **生产联调**：`default-provider=ESIGN_HTTP`，`http-provider.enabled=true` + 环境变量
- 出站鉴权头：`X-Scf-App-Id`、`X-Scf-Timestamp`、`X-Scf-Nonce`、`X-Scf-Signature`（HMAC-SHA256）
- 入站回调：沿用 EA-041 标准 `POST /integrations/contracts/sign-callback`

## 生产环境变量

```bash
SCF_CONTRACT_SIGN_DEFAULT_PROVIDER=ESIGN_HTTP
SCF_CONTRACT_SIGN_HTTP_PROVIDER_ENABLED=true
SCF_CONTRACT_SIGN_HTTP_BASE_URL=https://vendor.example.com
SCF_CONTRACT_SIGN_HTTP_APP_ID=your-app-id
SCF_CONTRACT_SIGN_HTTP_APP_SECRET=your-app-secret
SCF_CONTRACT_SIGN_CALLBACK_TOKEN=your-callback-secret
SCF_CONTRACT_SIGN_CALLBACK_VERIFICATION_MODE=TIMESTAMP_NONCE_SIGNATURE
```

## Smoke 场景（自动化）

1. `ESIGN_HTTP` 发起签署 → 内嵌 vendor 返回 `HTTP-FLOW-{taskId}` → 单证 `SIGNING`
2. HMAC 回调 `SUCCESS` → 单证 `SIGNED`
3. 按 ref 查单 + 主动 query-status → 供应商 `SUCCESS`
4. 配置页 `provider_connections` 展示 enabled/configured/base_url
5. vendor 鉴权失败 → `502 CONTRACT_SIGN_502`
6. 相同 callback nonce 第二次请求 → `403 AUTH_403`
