## 4. 标准供应商接口契约

### 发起签署

`POST /sign/create`

请求头：

- `X-Scf-App-Id`
- `X-Scf-Timestamp`
- `X-Scf-Nonce`
- `X-Scf-Signature = hex(HMAC_SHA256(appSecret, timestamp + "\n" + nonce + "\n" + rawBody))`

请求体核心字段：

- `task_id`
- `document_id`
- `file_id`
- `document_no`
- `business_type`
- `business_id`
- `signers`

响应体可为根对象或 `{ "data": ... }`：

```json
{
  "external_sign_ref": "REAL-SIGN-001",
  "provider_status": "ACCEPTED",
  "provider_message": "accepted"
}
```

### 主动查单

`GET /sign/status/{externalSignRef}`

响应体：

```json
{
  "external_sign_ref": "REAL-SIGN-001",
  "status": "COMPLETED",
  "signed_at": "2026-06-01T10:00:00Z",
  "failure_reason": null
}
```

状态标准化：

| 供应商状态 | 平台状态 |
|---|---|
| `ACCEPTED/PENDING/PROCESSING/SIGNING` | `PENDING_CALLBACK` 或 `PENDING` |
| `COMPLETED/SIGNED/SUCCESS` | `SIGNED` 或 `SUCCESS` |
| `FAILED/REJECTED/CANCELLED` | `FAILED` |

## 6. 剩余事项

1. 拿到真实供应商文档后，需要把字段映射从标准契约适配到厂商原生字段。
2. 如果供应商要求 RSA/SM2 而不是 HMAC，应新增具体厂商 Provider，不建议污染通用 HTTP Provider。
3. 真实联调时必须保存供应商原始响应摘要和 request id，便于对账与争议处理。
