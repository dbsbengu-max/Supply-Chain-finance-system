# EA-042 签章回调补偿池运营前端验收结果

## 1. 验收结论

通过。

Cursor 已完成签章回调补偿池的运营入口与前端闭环：合同签章配置页可跳转 Saga 运营中心，Saga 补偿任务支持签章回调复核快捷筛选、详情上下文展示、影响范围展示、跳转单证中心。Codex 复核后补充了后端 `@PathVariable` 显式参数名与供应商接入模板文档修正，并完成前后端验证。

| 范围 | 结论 | 说明 |

## 3. Codex 修正

1. `SagaOpsController` 的所有 `{id}` 路径变量补充 `@PathVariable("id")`，避免裸 `javac` 或未开启 `-parameters` 时出现参数名解析失败。
2. `docs/ESIGN_PROVIDER_TEMPLATE.md` 更新 EA-041 现状：
   - TOKEN 改为兼容模式。
   - TIMESTAMP_NONCE_SIGNATURE 改为生产推荐模式。
   - 补偿池落表改为 `biz_compensation_task`。
   - 验签失败明确不入补偿池，业务失败才进入人工复核池。

## 5. 剩余风险

1. 单证中心当前无法通过 `external_sign_ref` 自动定位单证，只能提示运营从列表打开合同单证并核对签署任务。若试点运营量上来，建议新增后端查询接口：`GET /documents/center/by-external-sign-ref/{externalSignRef}`。
2. 签章回调复核当前以 Saga 通用“重试/忽略/关闭”为主，尚未提供“主动查单”按钮。真实供应商接入后，建议基于 `ContractSignProvider.querySignStatus` 增加人工查单动作。
3. 前端构建仍存在 Element Plus 大 chunk 警告，与本次改动无直接关系，但上线前建议继续执行路由拆包优化。

## 6. 下一步建议

EA-043 建议推进“真实供应商接入前的签章对账能力”：

1. 后端新增按 `external_sign_ref` 查询签章任务和单证的接口。
2. Saga 补偿详情增加“主动查单”后端动作和前端按钮。
3. 增加 `CONTRACT_SIGN_STATUS_QUERY` 权限码与穿透测试。
4. 输出真实供应商接入 smoke：发起签署、成功回调、失败回调、未知回调入池、人工查单、忽略/关闭。
