# EA-038 合同单证中心与OCR复核流开发简报

日期：2026-05-31

## 1. 背景与决策

EA-037 已把 Saga 失败后的补偿闭环和人工处置台补齐，系统从“能跑主链路”进一步升级为“失败可解释、可处置、可审计”。

下一步建议推进 EA-038：**合同单证中心与OCR复核流**。原因是供应链金融系统的授信、放款、货权质押、凭证流转、清分兑付都依赖可信业务材料。当前系统已有文件上传、OCR、Excel导入、贸易订单单证接口，但还缺一个统一的“单证证据层”：

- 单证是否齐全；
- 单证是否属于当前项目/企业/业务对象；
- OCR识别结果是否经人工确认；
- 合同是否签署或上传；
- 融资、放款、货权、凭证操作前是否能引用可信单证。

EA-038 不做真实电子签章供应商对接，不做复杂合同模板生成；本期先建设可开发、可验收、可扩展的合同/单证中心。

## 2. 本期目标

1. 建立统一合同/单证列表，覆盖订单、代采、融资、仓储、凭证等业务对象；
2. 支持按业务类型维护必备单证清单，并能校验“缺失/待复核/已通过/已拒绝”；
3. 打通文件上传、OCR任务、人工复核、审计日志；
4. 在关键业务前置校验中预留单证校验能力，优先落地查询和校验接口；
5. 形成前后端可并行开发的 API、页面、权限、测试清单。

## 3. 范围边界

### 3.1 本期包含

| 能力 | 说明 |
|---|---|
| 单证中心 | 跨业务对象查询、筛选、查看详情、下载原文件、查看OCR结果 |
| 单证登记 | 复用 `/files/upload` 后登记业务单证，支持 `business_type`、`business_id` |
| 必备单证清单 | 按业务类型、产品类型、阶段维护必备/可选单证 |
| OCR复核 | 展示建议值、置信度、原文定位、人工确认/驳回 |
| 合同状态 | 记录合同上传、待签、已签、签署失败、作废等状态 |
| 单证校验 | 返回缺失项、待复核项、异常项，供融资/放款/质押前置校验调用 |
| 审计日志 | 上传、复核、驳回、状态变更均写审计 |

### 3.2 本期不包含

| 不包含项 | 处理方式 |
|---|---|
| 真实电子签章供应商对接 | 仅预留 `sign_provider`、`external_sign_ref` 字段 |
| 合同模板在线编辑 | 暂不实现，先上传PDF/图片/Office文件 |
| AI自动审批单证 | AI只给建议，关键金融动作必须人工确认 |
| 单证内容法律合规自动判定 | 仅做格式、必填字段、OCR置信度、业务一致性校验 |
| 大文件对象存储迁移 | 沿用现有文件上传能力，预留存储适配字段 |

## 5. 推荐API

| 方法 | 路径 | 权限 | 说明 |
|---|---|---|---|
| `GET` | `/documents/center` | `DOCUMENT_VIEW` | 单证中心列表 |

| `POST` | `/documents/center` | `DOCUMENT_UPLOAD` | 登记业务单证 |
| `POST` | `/documents/center/{id}/ocr` | `AI_OCR_EXECUTE` | 发起OCR |
| `POST` | `/documents/center/{id}/submit-review` | `DOCUMENT_REVIEW_SUBMIT` | 提交复核 |
| `POST` | `/documents/center/{id}/approve` | `DOCUMENT_REVIEW_APPROVE` | 复核通过 |
| `POST` | `/documents/center/{id}/reject` | `DOCUMENT_REVIEW_APPROVE` | 复核驳回 |
| `POST` | `/documents/center/{id}/archive` | `DOCUMENT_ARCHIVE` | 归档/作废 |
| `GET` | `/documents/requirements` | `DOCUMENT_REQUIREMENT_VIEW` | 必备单证规则列表 |

| `PUT` | `/documents/requirements/{id}` | `DOCUMENT_REQUIREMENT_MANAGE` | 修改必备单证规则 |
| `POST` | `/documents/validate` | `DOCUMENT_VIEW` | 校验业务对象单证齐备性 |

## 6. 前端页面建议

新增页面：`DocumentCenterView.vue`，路由：`/documents/center`。

| 区域 | 功能 |
|---|---|
| 顶部筛选 | 项目、业务类型、单证类型、状态、复核状态、合同状态 |
| 列表 | 单证编号、业务对象、单证类型、文件名、OCR置信度、复核状态、合同状态、更新时间 |
| 详情抽屉 | 文件预览、OCR结果、人工复核、审计时间线、关联业务 |
| 上传抽屉 | 选择业务类型/业务ID/单证类型，上传文件，选择是否OCR |
| 必备规则页签 | 维护业务阶段必备单证规则 |
| 校验面板 | 输入业务类型+业务ID+阶段，展示缺失/待复核/警告 |

交互约束：

- 复核通过前，关键字段显示“AI建议值”，不能作为已确认业务事实；
- 驳回必须填写原因；
- 合同作废、归档需二次确认；
- 跨项目单证不可见，后端必须重复鉴权；
- 文件下载、预览、OCR结果查看均应写审计或访问日志。

## 8. 测试验收

新增集成测试建议：

| 用例 | 验收 |
|---|---|
| 上传并登记订单单证 | 返回单证ID，列表可查询 |
| 发起OCR | 生成OCR任务并回写 `ocr_job_id` |
| OCR低置信度 | 校验接口返回 warning |
| 必备单证缺失 | `/documents/validate` 返回 `passed=false` |
| 单证复核通过 | 状态变为 APPROVED，写 review log 和 audit log |
| 单证驳回无原因 | 返回 400 |
| 跨项目访问 | 返回 403 或 404，不能泄露数据 |
| 普通会员管理必备规则 | 返回 403 |
| 归档后再次复核 | 返回 409 |
| `EA019RegressionRunner` | 纳入新增测试并全量通过 |

前端验收：

- `npm run build` 通过；
- 单证中心列表、上传抽屉、详情抽屉、复核动作、必备规则页签可用；
- 权限不足时按钮隐藏，后端仍返回 403；
- 金额、日期、状态标签与现有页面风格一致。

## 9. Cursor任务简报

### 任务目标

实现 EA-038：合同单证中心与OCR复核流，补齐统一单证列表、必备单证规则、OCR人工复核、单证校验接口和前端页面骨架。

### 技术约束

- 不改历史 Flyway 文件，只新增 `V1_1_029__document_contract_center.sql`。
- 优先复用现有 `/files/upload`、`/ai/ocr/jobs`、贸易订单单证能力。
- AI/OCR只能作为辅助建议，人工复核通过前不得驱动关键金融动作。
- 所有查询必须接入 `PermissionService` 和 `DataScopeHelper`。
- 上传、OCR、复核、驳回、归档必须写审计或 review log。

### 禁止事项

- 禁止把 OCR 结果直接写成最终确认值。
- 禁止真实调用电子签章供应商。
- 禁止删除历史单证和复核日志。
- 禁止把跨项目 403/404 的安全策略改回数据泄露式错误。
