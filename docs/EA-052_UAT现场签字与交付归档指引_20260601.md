## 3. 现场流程（约 45–60 分钟）

### Step 1 — 开场

- 说明：Mock/试点，非真实 vendor / 银行 / 生产灰度（见 EA-051 已知限制）
- 打开 `/uat/acceptance`，确认 M1–M12 列表可见

### Step 2 — 按 M1–M12 演示

- 严格按 [`DEMO_SCRIPT_M1-M12.md`](../deploy/pilot/delivery/DEMO_SCRIPT_M1-M12.md) 顺序
- **每步结束**在 UAT 页：
  - 状态：通过 / 未通过 / 跳过
  - 备注：现象、单号、截图文件名（如有）

### Step 3 — 导出现场签字包

1. 在 `/uat/acceptance` 点击 **导出签字包**
2. 保存为：`deploy/pilot/evidence/ea052-uat-archive/<场次>/UAT_SIGNOFF_EXPORT.md`
3. 若某步 fail：在备注写清复现步骤，并记入业务问题清单

### Step 4 — 自动化证据（技术侧）

```powershell
cd deploy\pilot\scripts
.\run-ea051-acceptance.ps1
```

或一键归档（含最新 A1–A4）：

```powershell
.\archive-ea052-uat.ps1 -RunAcceptance
```

### Step 5 — 业务问题清单

- 模板：[`BUSINESS_ISSUE_LOG_TEMPLATE.md`](../deploy/pilot/delivery/BUSINESS_ISSUE_LOG_TEMPLATE.md)
- 现场填写后保存到同一场次目录 `BUSINESS_ISSUES.md`

### Step 6 — 纸质/邮件签字

- 将导出 Markdown 中的签字区补全，或填写 [`EA-052_UAT签字包归档_20260601.md`](EA-052_UAT签字包归档_20260601.md)
- 业务 / 产品 / 技术 / 运维 四类角色 GO 或 NO-GO

## 5. Blocker 判定（现场）

| 级别 | 定义 | 处理 |
|---|---|---|
| **Blocker** | 主链路不可用、403 误拦管理员、数据错乱、无法登录 | 标记 fail，**NO-GO trial**，开缺陷单 |
| **Major** | 有 workaround，影响体验 | 记入问题清单，可 GO trial + 限期修复 |
| **Minor** | 文案、样式、非主路径 | 记入 backlog，不阻 GO trial |
